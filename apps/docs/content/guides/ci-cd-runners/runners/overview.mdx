Overview of the different runner types available in CI/CD Runners.

## Runner Types

CI/CD Runners support multiple runner types to meet different workload requirements:

- **Linux**: Native x64 and arm64 architectures
- **Windows**: Full Windows Server support
- **macOS**: Custom macOS runners
- **GPU**: NVIDIA and AMD GPU support for ML workloads
- **Pools**: Warm pools for faster job startup times

## Linux Runners

Linux runners support:

- x64 and arm64 architectures
- Various Linux distributions
- Docker and container support
- Flexible instance types

See [Linux Runners](/guides/ci-cd-runners/runners/linux) for details.

## Windows Runners

Windows runners support:

- Windows Server editions
- PowerShell and CMD
- Windows-specific tools
- .NET and Windows development

See [Windows Runners](/guides/ci-cd-runners/runners/windows) for details.

## macOS Runners

macOS runners support:

- macOS versions
- Xcode and iOS development
- macOS-specific tools
- Native macOS development

See [macOS Runners](/guides/ci-cd-runners/runners/macos) for details.

## GPU Runners

GPU runners support:

- NVIDIA GPUs
- AMD GPUs
- ML and AI workloads
- CUDA support

See [GPU Runners](/guides/ci-cd-runners/runners/gpu) for details.

## Runner Pools

Runner pools provide:

- Warm pools for faster starts
- Auto-scaling capabilities
- Cost optimization
- Load balancing

See [Runner Pools](/guides/ci-cd-runners/runners/pools) for details.
