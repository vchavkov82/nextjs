---
title: 'Remote Execution'
---

Execute Bazel builds remotely with CI/CD Runners for faster parallel builds.

## Overview

Remote execution allows Bazel to execute build actions on remote infrastructure, enabling parallel execution of actions and faster builds for large projects.

## How It Works

Remote execution:

- **Distributes actions**: Executes build actions on remote workers
- **Parallel execution**: Runs multiple actions simultaneously
- **Consistent environments**: Uses standardized build environments
- **Resource scaling**: Leverages high-performance compute resources

## Configuration

Configure remote execution in your `.bazelrc` file:

```bazelrc
build --remote_executor=https://exec.runners.dev
build --remote_header=x-api-key=$EXEC_TOKEN
build --remote_instance_name=my-project
```

## GitHub Actions Integration

Use remote execution in GitHub Actions:

```yaml
jobs:
  build:
    runs-on: self-hosted
    steps:
      - uses: actions/checkout@v3
      - name: Configure Bazel Remote Execution
        run: |
          echo "build --remote_executor=${{ secrets.BAZEL_EXEC_ENDPOINT }}" >> .bazelrc
          echo "build --remote_header=x-api-key=${{ secrets.BAZEL_EXEC_TOKEN }}" >> .bazelrc
          echo "build --remote_instance_name=${{ github.repository }}" >> .bazelrc
      - name: Build with Remote Execution
        run: bazel build //...
```

## Platform Configuration

Configure execution platforms:

```bazelrc
build --platforms=@platforms//os:linux
build --host_platform=@platforms//os:linux
build --extra_execution_platforms=@platforms//os:linux
```

## Benefits

Remote execution provides:

- **Parallel builds**: Execute many actions simultaneously
- **Faster builds**: Leverage high-performance compute
- **Scalability**: Handle large builds efficiently
- **Resource efficiency**: Use specialized compute resources
- **Consistent environments**: Standardized build environments

## Best Practices

- Use appropriate execution platforms
- Configure resource limits
- Monitor execution performance
- Use execution pools for cost optimization
- Set appropriate timeouts

## Resource Configuration

Configure execution resources:

```bazelrc
build --remote_executor=https://exec.runners.dev
build --remote_header=x-api-key=$EXEC_TOKEN
build --jobs=100
build --remote_timeout=3600
```

## Troubleshooting

Common issues:

- **Execution failures**: Check executor availability and configuration
- **Timeout errors**: Increase timeout values or optimize actions
- **Resource limits**: Adjust resource requests
- **Authentication errors**: Verify execution token is valid
