Overview of using CI/CD Runners with Gitea.

## Overview

CI/CD Runners provide optimized support for Gitea workflows, allowing you to run Gitea Actions on self-hosted infrastructure with improved performance and cost savings.

## Features

CI/CD Runners support for Gitea includes:

- **Gitea Actions Integration**: Run Gitea Actions workflows on CI/CD Runners
- **Self-Hosted Support**: Works with self-hosted Gitea instances
- **Scalable Runners**: Scale runners based on workload demands
- **Multiple Platforms**: Support for Linux, Windows, and macOS runners
- **Caching Strategies**: Optimized caching for Gitea Actions jobs
- **Security**: Secure runner registration and job execution

## Benefits

Using CI/CD Runners with Gitea provides:

- **Cost savings**: More cost-effective than shared runners
- **Performance**: Customizable compute resources for faster builds
- **Control**: Full control over runner environment and dependencies
- **Scalability**: Scale runners based on project needs
- **Privacy**: Keep sensitive builds on your own infrastructure
- **Flexibility**: Support for custom images and configurations

## Quick Start

Get started with Gitea Actions on CI/CD Runners:

1. [Set up Gitea Actions](/guides/ci-cd-runners/gitea/setup) on your Gitea instance
2. [Configure runners](/guides/ci-cd-runners/gitea/runner-configuration) for your workflows
3. [Create Gitea Actions workflows](/guides/ci-cd-runners/gitea/workflows) to use your runners

## Use Cases

CI/CD Runners with Gitea are ideal for:

- Self-hosted Gitea instances requiring dedicated runners
- Projects with specific dependencies or requirements
- Large teams with high CI/CD workload
- Sensitive projects requiring private infrastructure
- Cost optimization for high-volume CI/CD usage
- Custom build environments and configurations

## Gitea Actions

Gitea Actions is a CI/CD system integrated into Gitea. CI/CD Runners can execute Gitea Actions workflows, providing the same workflow syntax as GitHub Actions.
